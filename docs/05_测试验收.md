# 差旅搭子（Travel Buddy）— 阶段五：测试验收

> 文档定位：本文件为「差旅搭子」桌面软件上线前的测试验收交付物，覆盖功能测试、边界场景、异常场景、回归测试、Bug 规范、上线阻断标准与验收清单。
> 适用版本：阶段五验收基线（Python 3.11 + PySide6 + SQLite）
> 读者：QA、开发、产品经理、项目负责人

---

## 0. 测试范围与策略总览

| 维度 | 说明 |
| --- | --- |
| 测试对象 | 差旅搭子桌面客户端（Windows / macOS / Linux） |
| 测试类型 | 功能测试、边界测试、异常测试、回归测试、验收测试 |
| 测试环境 | 开发自测 + QA 验收环境 + 预发布（Staging）环境 |
| 核心链路 | 采集 → 解析 → 入库 → 管理 → 仪表盘 → 导出 |
| 通过标准 | 全部 P0/P1 用例通过，P2 通过率 ≥ 95%，无 P0 遗留缺陷 |
| 回归范围 | 见第 5 节「回归测试范围」 |

优先级约定：
- **P0（致命/阻断）**：核心链路不可用、数据丢失/错乱、无法上线。
- **P1（严重）**：主要功能缺陷、 workaround 成本高、必须修复。
- **P2（一般）**：次要缺陷、可带病发布或下版本修复。

## 3. 上线阻断判断标准（判定清单）

### 3.1 必须阻断上线（任一命中即阻断）

- [ ] **数据正确性**：解析金额、税额、税号、日期出现系统性错误（如 `价税合计 ≠ 不含税 + 税额`）。
- [ ] **数据丢失/损坏**：导入/编辑/导出导致已有发票数据丢失、错乱或 SQLite 库损坏且无恢复手段。
- [ ] **核心链路阻断**：采集、解析、管理、导出任一主流程在验收环境必现不可用。
- [ ] **阻断性异常未捕获**：邮件登录失败、网络中断、文件损坏、磁盘满等场景导致客户端崩溃/闪退。
- [ ] **重复/串数据**：重复发票未去重，或导出串入非筛选范围数据导致报销错误。
- [ ] **安全/隐私**：本地数据明文泄露风险、附件越权访问、IMAP 凭据明文落盘。
- [ ] **P0 缺陷遗留**：存在任意未修复且无法 workaround 的 P0 缺陷。
- [ ] **验收未达标**：P0/P1 用例未 100% 通过，或 P2 通过率 < 95%。

### 3.2 可带病发布（需产品/负责人确认并记录）

- [ ] **P2 次要 UI 问题**：个别排版错位、非关键文案、低分辨率下显示瑕疵。
- [ ] **偶现且可绕过**：复现概率极低（<5%）且有明确 workaround 的非核心功能问题。
- [ ] **非核心增强**：仪表盘某趋势视图、附件预览某格式等次要功能暂不完整。
- [ ] **已知第三方限制**：特定扫描件无 OCR 导致解析失败（已在 PRD 标注为预期外）。
- [ ] **性能非阻断**：超大文件处理较慢但未超时崩溃，已给出进度提示。
- [ ] **文档/提示类**：帮助文档、空状态引导文案待优化。

> 带病发布须登记《上线风险清单》（见第 5 节），并由 QA 负责人 + 产品经理双签确认。

| RK-05 | SQLite 并发写入竞争 | 异常 | 低 | 高 | 事务锁 + 写入失败提示 | 阻断（须修复） |

> 说明：处置结论为「阻断」的项未关闭前不得上线；「带病」项须双签并持续跟踪。

## 7. 项目规范提醒

- 测试过程中发现的 Bug，请使用第 2 节模板创建**事项（Issue）**，并**指派给对应开发同学**。
- 每个 Bug 事项须**关联回原需求或 PRD 文档**（建议关联 `docs/02_PRD.md` 对应章节及第 1 节相关用例编号），保证缺陷可追溯、需求可闭环。
- 带病发布与上线阻断判定，须经 QA 负责人与产品经理双签，并登记《上线风险清单》。
