### 1.4 风险点（技术 / 数据 / 合规）

**技术风险**
- R-T1 PDF 版式多样性导致解析字段抽取不准（火车票/增值税专票/电子普票结构差异大）。→ 解析器分层 + 置信度 + 标黄人工复核；建立解析样例回归集。
- R-T2 批量导入（20+ 文件）阻塞 UI。→ 解析放 `QThread`/`QRunnable`，进度回调，禁止主线程同步 IO。
- R-T3 Qt 打包体积与启动慢（PySide6 + pandas + pdfplumber 合计 >200MB）。→ 评估 `--onefile` vs `--onedir`，按需裁剪隐式依赖。

**数据风险**
- R-D1 去重误删：仅按 `发票代码+号码` 去重，手工票无代码时可能重复或误合并。→ 唯一索引仅对「非空代码+号码」生效（部分唯一索引），手工票允许重复并提示。
- R-D2 附件路径失效：SQLite 存绝对路径，用户改数据目录后附件打不开。→ 附件表存「相对 DATA_DIR 的相对路径」，访问时拼接当前 DATA_DIR；迁移时整体拷贝。
- R-D3 单文件库写入锁：并发导入线程写同库可能 `database is locked`。→ 单写者队列（写操作串行化），读用 WAL 模式。

**合规风险**
- R-C1 IMAP 授权码泄露（明文落盘）。→ Q-4 决议加密存储；不在日志打印。
- R-C2 税务敏感数据（税号、金额）本地明文。→ 提供「数据库本地加密 / 应用锁」开关（阶段一可仅做应用启动口令）。
- R-C3 隐私承诺「不上云」需产品侧明确，且链接下载/邮件导入均不出本机。→ 代码中禁用任何遥测/外传；设置页展示「数据均存于本机」。

### 2.3 数据流

```
[来源] 本地/邮件/链接/手工
   │
   ▼  Importer.discover() → SourceItem 列表
   ▼  Importer.acquire() → 落地到 attachments 临时区（rel_path 计算）
   ▼  Parser.parse(file) → ParseResult{fields, amount, tax, vendor, dates, category_key, confidence, warnings}
   ▼  classifier.classify() → category_id（命中 DEFAULT_CATEGORIES）
   ▼  组装 Invoice（含 need_review = confidence < 阈值）
   ▼  dedup.check(invoice) → 命中唯一索引则 skip（ImportResult.skipped=True）
   ▼  InvoiceRepo.save() → 写队列串行化入库（WAL）
   ▼  触发 UI 列表/仪表盘刷新（信号槽）
        │
        ├─ 管理页：搜索/筛选/排序（Repos 查询）
        ├─ 详情页：编辑字段 → save（updated_at 刷新）
        ├─ 行程页：关联 trip_id
        └─ 导出页：Exporter.export(selected) → Excel/PDF（+附件打包）
```

关键不变量：
- 附件永远先于发票落盘；删发票级联删附件（FK `ON DELETE CASCADE`）。
- 所有写操作经单写者队列，避免 `database is locked`。
- 解析失败不阻断导入：写入 `need_review=1` 的草稿，交给用户补录。

### 2.4 关键接口约定（方法签名建议）

#### 2.4.1 导入器接口（importers/base.py）
```python
from abc import ABC, abstractmethod
from dataclasses import dataclass, field
from typing import Optional, Callable, List

@dataclass
class SourceItem:
    uri: str                 # 文件绝对路径 / 邮件 UID / URL
    display_name: str = ""
    meta: dict = field(default_factory=dict)

@dataclass
class ImportResult:
    file_path: str
    ok: bool = False
    skipped: bool = False    # 去重跳过
    invoice_id: Optional[int] = None
    error: Optional[str] = None
    confidence: float = 1.0

class Importer(ABC):
    source: str = "manual"                       # 与 config.SOURCE_* 对齐

    @abstractmethod
    def discover(self) -> List[SourceItem]:
        """枚举待导入源（本地多选文件 / 邮件列表 / 单个 URL）。"""

    @abstractmethod
    def acquire(self, item: SourceItem) -> str:
        """将源落地为本地文件，返回本地绝对路径；失败抛 ImporterError。"""

    def run(self,
            items: List[SourceItem],
            on_progress: Callable[[int, int, str], None] = None) -> List[ImportResult]:
        """编排：acquire→parse→dedup→save；逐条回调 (done, total, name)。基类提供默认实现，子类可覆写。"""
```

具体实现约定：
- `LocalImporter.discover()` 接收 `QFileDialog` 多选结果（路径列表）。
- `EmailImporter.discover()` 用 `imaplib` 拉近 N 天带附件邮件，返回 `SourceItem(uri=mail_uid, meta={subject,date})`；`acquire()` 下载附件落盘。
- `UrlImporter.discover()` 接收单/多个 URL；`acquire()` 用 `requests` 下载（带 10s 超时）。
- `ManualImporter` 不走文件流，直接 `run()` 接收表单 `Invoice` 入库。

#### 2.4.2 解析器接口（parsers/base.py）
```python
@dataclass
class ParseResult:
    fields: dict                 # 原始抽取字段
    amount: float = 0.0
    tax_amount: float = 0.0
    amount_excl_tax: float = 0.0
    issue_date: str = ""
    vendor_name: str = ""
    vendor_tax_id: str = ""
    invoice_code: str = ""
    invoice_number: str = ""
    category_key: str = "other"  # 归类建议
    confidence: float = 1.0      # 0~1
    warnings: List[str] = field(default_factory=list)

class Parser(ABC):
    supported_types: tuple = ()          # (".pdf",) 等

    @abstractmethod
    def parse(self, file_path: str, file_type: str) -> ParseResult:
        """抽取字段；无法抽取时返回 confidence 低 + warnings，不抛异常。"""

    def confidence_of(self, result: ParseResult) -> float:
        """按关键字段缺失度计算置信度（可被基类共用）。"""
        required = ["amount", "invoice_number", "vendor_name"]
        hit = sum(1 for k in required if getattr(result, k, None))
        return round(hit / len(required), 2)
```

- `PdfParser`：`pdfplumber.open()` 抽文本 → 正则（发票代码 ` No\.?(\d{12})`、价税合计、税额、税号 `税号[:：]?\s*([A-Z0-9]{15,20})` 等）→ 填充 `ParseResult`。
- `classifier.classify(text, result)`：关键词命中（"火车"/"高铁"→train，"酒店"/"住宿"→hotel，"餐"→meal，"出租车"/"网约车"→taxi）→ 返回 `category_key`；未命中→`other`。
- `ImageParser` / `OfdParser`：阶段一 `raise NotImplementedError`（接口预留，满足 Q-1/Q-5）。

#### 2.4.3 导出器接口（exporters/base.py）
```python
@dataclass
class ExportOptions:
    fields: List[str] = field(default_factory=list)   # 选中导出字段，空=全部
    include_attachments: bool = False                 # 是否打包原始附件
    template: str = ""                                # 自定义模板路径（Q-3）
    group_by: str = ""                                # trip/category/status/none

@dataclass
class ExportResult:
    ok: bool
    dest: str = ""
    count: int = 0
    package: str = ""     # 附件打包 zip 路径（若有）
    error: Optional[str] = None

class Exporter(ABC):
    fmt: str = "xlsx"     # xlsx / pdf

    @abstractmethod
    def export(self, invoices: List[Invoice], dest: str, options: ExportOptions) -> ExportResult:
        """生成报销单；invoices 由调用方按筛选结果传入。导出后不自动改状态（状态变更由 UI 显式确认，见 S4）。"""
```

- `ExcelExporter`：`pandas.DataFrame` 组装 → `openpyxl` 写样式（表头加粗、金额两位、列宽自适应）。
- `PdfExporter`：封面（汇总金额/张数/导出人/日期）+ 明细表；如用 reportlab 需自行处理 CJK 字体（见 §4.4）；如用 Qt `QPrinter` 可直接复用界面字体。

#### 2.4.4 持久层仓储接口（db/repositories.py，摘要）
```python
class InvoiceRepo:
    def save(self, inv: Invoice) -> int: ...           # 返回 id；唯一索引冲突则抛 DedupError
    def get(self, id: int) -> Invoice: ...

    def update(self, inv: Invoice) -> None: ...
    def delete(self, id: int) -> None: ...             # 级联附件
    def set_status(self, ids: List[int], status: str) -> None: ...

class TripRepo / CategoryRepo / AttachmentRepo / SettingsRepo:  # 各自 CRUD + 批量关联
```
`InvoiceFilter` 建议字段：`status, category_id, trip_id, keyword, date_from, date_to, source`。

### 3.2 关键页面状态清单（空 / 加载 / 错误 / 空数据）

| 页面 | 空态 Empty（无数据首次） | 加载 Loading | 错误 Error | 空数据 NoResult（有数据但筛选无命中） |
|---|---|---|---|---|
| 主窗口 | 首次启动引导卡「点击导入你的第一张发票」 | 全局细粒度加载条（导入/导出时） | 数据库打开失败弹窗 + 重试 | — |
| 仪表盘 | 「还没有发票，导入后看这里」插画 + 导入入口 | 图表骨架屏 | 统计查询异常提示「统计暂不可用」 | 「暂无符合筛选的数据」 |
| 发票列表 | 空态插画 + 「从本地/邮件导入」 | 列表骨架 + 行占位 | 加载失败「无法读取发票，检查数据库」 | 「当前筛选无结果，清除筛选」 |
| 发票详情 | —（必由列表进入） | 字段占位 | 附件读取失败「原图已移动」 | — |
| 导入向导 | 步骤 1 待选择高亮 | 解析中转圈 + 逐条进度「3/20」 | 某文件解析失败标红但不中断；邮箱连接失败提示检查授权码 | 去重跳过提示「X 张已存在，已跳过」 |
| 行程管理 | 「新建第一个行程」引导 | 保存中转圈 | 行程保存失败「名称重复/必填缺失」 | 行程下无发票「该行程还没挂发票」 |
| 设置 | 默认值预填 | 测试连接中转圈 | 连接测试失败「主机/端口/授权码错误」 | — |

> 通用规则：所有 Loading 不许阻塞主线程（走后台线程）；Error 必须给出**可操作下一步**（重试/检查/去设置）；Empty/NoResult 必须有**主行动按钮**（导入/清除筛选/新建）。

### 3.3 交互边界（明确哪些不做 / 何时禁用）

- 删除发票：二次确认弹窗（含「同时删除本地附件」勾选，默认勾选）。
- 状态流转：仅允许合法迁移（`draft→pending→submitted→reimbursed`，`submitted→rejected→pending`）；`reimbursed` 不可直接回 `draft`。非法操作按钮置灰。
- 批量操作上限：单次勾选导出建议 ≤ 500 张，超出提示分批（防内存/打包过大）。
- 编辑中的发票：未保存离开详情页弹「放弃修改？」。
- 邮件导入：仅拉取带 PDF/图片/OFD 附件的邮件；不删服务器邮件；默认近 30 天可配。
- 链接下载：仅允许 http/https；禁止重定向到内网/本机地址（防 SSRF，轻量校验 host）。
- 解析失败项：进入列表 `need_review=1` 标黄，不可一键「全部确认入库」（必须人工过一遍）。

### 3.4 设计验收点（Design Acceptance）

1. 三维筛选（行程/分类/状态）可任意组合且结果即时刷新（<200ms 感知）。
2. 四态（空/加载/错误/空数据）在每个页面均有明确文案 + 主行动按钮，无「白屏/死界面」。
3. 发票卡片一眼可读：分类图标 + 金额 + 状态徽标 + 日期 + 来源标签；需复核项视觉醒目（黄边/角标）。
4. 导入向导步骤可回退、可中断，失败项不阻断整批。
5. 仪表盘图表在中文环境下无「方块/乱码」（字体验收，见 §4.4）。
6. 关键敏感操作（删除、提交、改状态）均有确认或禁用态，无误触。
7. 设置页明确展示「所有数据仅存于本机：<DATA_DIR>」，落实隐私承诺（R-C3）。

## 4. 研发视角

### 4.1 技术依赖与版本基线

- 锁定：`python==3.11`、`PySide6>=6.6`、`pdfplumber>=0.10`、`pandas>=2.0`、`openpyxl>=3.1`、`requests>=2.31`。
- 建议 `requirements.txt` + `requirements-dev.txt`（pytest、pyinstaller 仅 dev）。
- 解析样例回归集：在 `tests/fixtures/` 放 20+ 真实版式发票（火车/机票/酒店/打车/餐饮/增值税专票），CI 跑解析准确率 ≥95%（对应需求定量目标）。

### 4.2 接口问题（研发侧待对齐）

- I-1 `Importer.run` 默认实现放 `services/import_service.py`，避免各 importer 重复写解析/去重逻辑（单一职责）。
- I-2 `Parser.parse` 约定「不抛异常、低置信度返回 warnings」，保证导入向导能逐条标红而不中断。
- I-3 进度回调签名统一 `on_progress(done, total, name)`，UI 用 `QMetaObject.invokeMethod` 或信号跨线程刷新。
- I-4 `Exporter.export` 不隐式改状态；状态变更由 UI 显式调用 `InvoiceRepo.set_status`，避免「导出即提交」误判（对应场景 S4）。
- I-5 `rel_path` 存取统一走 `utils/path_utils.py`，禁止在业务代码里拼绝对路径（解 R-D2）。

### 4.3 数据结构注意事项

- `amount/tax_amount/amount_excl_tax` 用 `REAL`；若后续涉及报销精度争议，迁移为「分」整数存储（预留）。
- 冗余字段（`category_name/category_icon/trip_name`）为查询性能冗余，更新分类/行程名时需同步刷新（在 Repo 更新时级联写）。
- `settings` 用键值表，读取做类型转换（`value_type`）；敏感键（`imap_pass`）只存密文。
- 所有时间字段统一 `datetime.now().strftime("%Y-%m-%d %H:%M:%S")`，避免时区混乱（单机场景按本地时区即可，但需文档化）。

### 4.4 跨平台兼容性（Windows / macOS / Linux）

| 维度 | 风险 | 方案 |
|---|---|---|
| 路径分隔符 | Win 用 `\`、类 Unix 用 `/` | 全程 `pathlib.Path` / `os.path.join`；DB 只存相对路径 `rel_path` |
| 数据目录 | 各平台用户目录不同 | `config.default_data_dir()` 用 `os.path.expanduser("~")`，已跨平台；macOS 可放 `~/Library/Application Support/差旅搭子`（后续优化） |
| 中文字体（图表） | matplotlib 默认无 CJK，Linux 常缺字体→方块 | `utils/font_utils.py`：运行时探测并注册字体（Win: `C:/Windows/Fonts/msyh.ttc`；macOS: `/System/Library/Fonts/PingFang.ttc`；Linux: `WenQuanYi`/`Noto Sans CJK`），设置 `plt.rcParams["font.sans-serif"]`；Qt 控件用 `QFont` 设回退 |
| IMAP SSL | 端口/SSL 模式差异（QQ 需 SSL 993） | `COMMON_IMAP_HOSTS` 已含主机；增加 `imap_ssl` 开关与连接测试；失败给具体错误 |
| 打包 | PyInstaller 各平台隐藏导入不同 | 三平台分别出包；`hooks` 处理 `pdfplumber`/`pandas` 隐式依赖；`--onedir` 优先保兼容 |
| 文件关联/通知 | macOS 沙盒限制 | 阶段一不申请沙盒，本地文件直接访问；链接下载注意 macOS 全磁盘访问权限提示 |

### 4.5 实现风险与缓解（研发侧）

| 风险 | 缓解 |
|---|---|
| PDF 扫描件无文本层 → 解析全失败 | `PdfParser` 检测文本长度，过短则 `confidence=0` + warning「疑似图片 PDF，请走手工/后续 OCR」 |
| 批量导入卡 UI | 解析/下载全部进 `QRunnable` 线程池，主线程仅更新模型 |
| 并发写库锁 | 单写者队列 + `PRAGMA journal_mode=WAL` + 写超时重试 |
| 附件路径失效（迁移/改目录） | 存相对路径 + 迁移工具整体拷贝 `DATA_DIR` |
| 邮件授权码泄露 | `crypto_utils` 加密存储，禁用日志打印；优先 `keyring`，回退本地加密文件 |
| 导出大批量内存暴涨 | 分页查询 + 流式写；附件打包用 `zipfile` 增量 |
| 解析准确率回归 | `tests/` 固定样例集 + CI 断言关键字段准确率 ≥95% |
